<span data-ttu-id="60ae6-p102">SharePoint Server 2016 offre la possibilità per autenticare gli utenti che utilizzano l'autenticazione basata sulle attestazioni, semplificando la gestione degli utenti tramite l'autenticazione di essi con i provider di identità diversi che si considera attendibile, ma un utente di gestione. Ad esempio, invece di gestire l'autenticazione degli utenti tramite servizi di dominio Active Directory (AD DS), è possibile consentire agli utenti per l'autenticazione con Azure Active Directory (Azure Active Directory). In questo modo l'autenticazione per gli utenti di cloud-only con il suffisso onmicrosoft.com il proprio nome utente, gli utenti sincronizzati con una directory locale e utenti guest da altre directory invitati. This enables authentication for cloud-only users with the onmicrosoft.com suffix in their username, users synchronized with an on-premises directory, and invited guest users from other directories. It also enables you to take advantage of Azure AD features such as multi-factor authentication and advanced reporting capabilities.</span></span>

## <a name="step-6-add-a-saml-11-token-issuance-policy-in-azure-ad"></a><span data-ttu-id="60ae6-231">Passaggio 6: Aggiungere un criterio di emissione di token SAML 1.1 in Azure Active Directory</span><span class="sxs-lookup"><span data-stu-id="60ae6-231">Step 6: Add a SAML 1.1 token issuance policy in Azure AD</span></span>

<span data-ttu-id="60ae6-p121">Quando si crea l'applicazione di Azure Active Directory nel portale, l'impostazione predefinita all'utilizzo di SAML 2.0. Il codice seguente è necessario scaricare gli [esempi che illustrano l'interazione con Azure Active Directory grafico](https://github.com/kaevans/spsaml11/tree/master/scripts).</span><span class="sxs-lookup"><span data-stu-id="60ae6-p121">When the Azure AD application is created in the portal, it defaults to using SAML 2.0. ```

