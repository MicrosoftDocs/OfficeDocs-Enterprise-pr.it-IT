## <a name="what-does-end-of-support-mean"></a>Quali fine consente di supporto Media?

Prodotti Microsoft, ad esempio PerformancePoint Server 2007 SP3, il software ProClarity e SharePoint Server 2007 SP3 con un ciclo di vita del supporto durante il quale Microsoft sono disponibili nuove funzionalità, correzioni e aggiornamenti di sicurezza. Il ciclo di vita di un prodotto in genere dura per 10 anni dalla data di rilascio iniziale del prodotto e la fine del ciclo di vita è noto come fine del prodotto del supporto tecnico. Come ProClarity, PerformancePoint Server e SharePoint Server 2007 hanno raggiunto il fine del supporto, Microsoft non fornirà più:
  
- Supporto tecnico per i problemi che possono verificarsi
    
- Correzioni degli errori per i problemi che vengono individuate e che possono influenzare la stabilità e l'usabilità dei server
    
- Le patch di protezione per le vulnerabilità che vengono individuate e che potrebbe essere applicazioni o server vulnerabili ad attacchi di violazioni della protezione
    
- Aggiornamenti di fuso orario
    
L'installazione di PerformancePoint Server 2007 SP3, SharePoint Server 2007 SP3 e ProClarity continua a eseguire anche se è terminata in supporto. Tuttavia, si consiglia di eseguire la migrazione da queste applicazioni più presto possibile.

### <a name="use-power-bi-in-the-cloud-or-on-premises"></a>Utilizzare Power BI (nel cloud o locale)

Power BI è una serie di strumenti di business analitica per analizzare i dati e condividere informazioni. Utilizzare Power BI per creare rapporti e dashboard utilizzando locali o le origini dati in linea. Gli utenti possono visualizzare e utilizzare i rapporti e dashboard con computer o dispositivi mobili.
  
Power BI non è incluso in Office 365 o SharePoint Server, ma un separato oltre che Power BI Desktop, Power BI gateway e il servizio di Power BI. Power BI inoltre si integra con SharePoint Online. È possibile iniziare a creare Power BI gratuito e a seconda dei dati business e utilizzo delle esigenze, eseguire l'aggiornamento a Power BI Pro. Per ulteriori informazioni, vedere [What ' s Power BI?](https://go.microsoft.com/fwlink/?linkid=841341)
  
### <a name="use-reporting-services-on-premises"></a>Utilizzo di Reporting Services (locale)

SQL Server Reporting Services offre una soluzione report affidabile, oltre alla possibilità di installare e configurare Reporting Services in modalità nativa o in modalità integrata SharePoint. È possibile creare relazioni utilizzando diversi strumenti, tra cui progettazione Report e Generatore Report Power View. Con l'ultima versione di SQL Server, è inoltre possibile utilizzare pubblicazione Report di SQL Server Mobile per fornire rapporti di scalabilità orizzontale per le dimensioni dello schermo, offrendo la possibilità di utilizzare i report nei rispettivi dispositivi mobili dell'organizzazione. Per ulteriori informazioni, vedere [SQL Server Reporting Services (SSRS): creare, distribuire e gestire le relazioni per dispositivi mobili e impaginate](https://go.microsoft.com/fwlink/?linkid=841342).
  
### <a name="use-performancepoint-services-on-premises"></a>Usare PerformancePoint Services (locale)

Come è noto, PerformancePoint Server 2007 è stato acquistato separatamente da SharePoint Server 2007. A partire da SharePoint Server 2010, PerformancePoint Services è un'applicazione di servizio in SharePoint Server. Ciò significa che non è necessario acquistare hardware o licenze server separato per poter utilizzare PerformancePoint Services.
  
Per spostare PerformancePoint Services di PerformancePoint Server 2007, passare a una versione più recente di SharePoint Server e configurazione di PerformancePoint Services. La versione del Server di SharePoint a cui si sta spostando consente di determinare se è possibile importare il contenuto esistente del dashboard di PerformancePoint Server 2007 a PerformancePoint Services.
  
- Se sta eseguendo un aggiornamento a SharePoint Server 2010, è possibile importare il contenuto di PerformancePoint dashboard di PerformancePoint Server 2007 a PerformancePoint Services in SharePoint Server 2010. Per ulteriori informazioni sul funzionamento, vedere [Importazione guidata: contenuto di PerformancePoint Server 2007 a SharePoint Server 2010](https://go.microsoft.com/fwlink/?linkid=838873).
    
- Se si sta spostando per SharePoint Server 2013 o SharePoint Server 2016, sarà probabilmente necessario creare un nuovo contenuto dashboard (origini dati, report, scorecard e pagine di dashboard).
    
Per iniziare il piano di aggiornamento di PerformancePoint Services, vedere le risorse seguenti:
  
1. [SharePoint Server 2007 fine del supporto Roadmap](sharepoint-2007-end-of-support.md)
    
2. Quando si stabilisce quale versione di SharePoint a cui si sta spostando, vedere l'articolo corrispondente per PerformancePoint Services:
    
  - [Pianificare PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841363)
    
  - [Panoramica di PerformancePoint Services in SharePoint Server 2013](https://go.microsoft.com/fwlink/?linkid=841551)
    
  - [Panoramica di PerformancePoint Services in SharePoint Server 2016](https://go.microsoft.com/fwlink/?linkid=874704)
    
Quando esegue l'aggiornamento a PerformancePoint Services, sarà possibile accedere a diverse nuove funzionalità e miglioramenti. PerformancePoint Services offre scorecard migliorata, nuove visualizzazioni, ad esempio l'albero di scomposizione e report dettagli KPI e ulteriori tipi di grafico, migliori funzionalità di filtraggio Time Intelligence e conformità di migliorare l'accessibilità. Per ulteriori informazioni, vedere [Novità di PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841343).
